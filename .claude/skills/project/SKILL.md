## Purpose

Orchestrate end-to-end project management workflows using the `project-manager` orchestrator to coordinate progress tracking, agent work synthesis, and stakeholder communication across complex multi-agent implementations.

## Variables

ACTION: $1 (track, report, plan-update, coordinate, review)
SCOPE: $2 (current-phase, all-phases, specific-agents, deliverables)
REPORT_OUTPUT_DIR: `plans/<plan-name>/reports/project-report.md`

## Workflow:

Use the `project-manager` orchestrator to:
- Coordinate comprehensive progress tracking across all agent workstreams
- Orchestrate collection and synthesis of agent reports and deliverables
- Manage stakeholder communication and status reporting workflows
- Execute resource allocation and dependency resolution
- Synthesize project status into actionable management intelligence

**How to prompt the orchestrator:**
- Launch `project-manager` orchestrator with specific management objectives and scope
- Configure parallel progress monitoring across multiple workstreams and agents
- Set expectations for comprehensive status synthesis and risk identification
- Request actionable management recommendations with resource planning

**Project coordination requirements:**
- **IMPORTANT:** Provide consolidated status across all workstreams and dependencies
- **IMPORTANT:** Include risk assessment with mitigation strategies
- **IMPORTANT:** Generate actionable resource allocation recommendations
- **IMPORTANT:** Maintain clear audit trails of decisions and priority changes
- **IMPORTANT:** Synthesize complex project data into executive-friendly reporting
